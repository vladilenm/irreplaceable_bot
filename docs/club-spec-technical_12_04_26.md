# КЛУБ НЕЗАМЕНИМЫХ — Техническая спецификация

---

## 1. ЧТО ЭТО

Закрытое подписочное Telegram-сообщество для профессионалов, строящих персональные AI-системы из агентов. Формат — живая среда с горизонтальным peer-to-peer тоном, без иерархии «учитель → ученик».

**Ключевая философия:** Система > Навык.

**Формат:** каста, а не курс. Нет финишной линии — среда растёт вместе с участником.

## 2. АРХИТЕКТУРА ПРОДУКТА

### 2.1. Спринтовая система (90-дневная траектория)

| Фаза | Спринт | Формат | Результат участника | Статус |
|------|--------|--------|---------------------|--------|
| **Инициация** (1–14 дни) | Спринт 1 | 7-дневный спринт → первый агент | 1 работающий AI-агент под конкретную задачу | Сборщик |
| **Усиление** (15–45 дни) | Спринт 2 «Штаб» | 14-дневный бот-спринт, 10 сообщений | 3 агента в связке: Производитель / Контролёр / Аналитик | Архитектор |
| **Прикладное + Оркестрация** (45–90 дни) | Спринт 3 «Оркестратор» | 8 шагов в Mini App, без привязки к дням | Claude Project с оркестратором, метрика «было → стало» | Оператор |
| **Следующий** (90+ дни) | Спринт 4 | Тема по голосованию участников | TBD | TBD |

### 2.2. Структура Спринта 2 «Штаб»

**Механика:** Telegram-бот отправляет сообщения каждые 2–3 дня.

**Каждое сообщение бота — 4 элемента:**

1. Контекст (1 предложение) — где участник сейчас
2. Задание (2–3 предложения) — что делать
3. Ресурс (опционально) — ссылка на промпт/фреймворк
4. Дедлайн + действие — результат в чат до [дата]

**Таймлайн:**

| День | Тип | Содержание |
|------|-----|-----------|
| 1 | Задание | Диагностика первого агента + карта штаба (три роли) |
| 3 | Направление | Логика цепочки: как роли связаны |
| 4 | Задание | Создай Агента №2 — первую недостающую роль |
| 6 | Пинг | Промежуточное напоминание |
| 7 | Чекпоинт | Покажи связку двух агентов с ролями |
| 8 | Задание | Создай Агента №3 — последнюю роль |
| 10 | Пинг | Промежуточное напоминание |
| 11 | Чекпоинт | Покажи полную тройку |
| 12 | Задание | Свяжи трёх агентов в цепочку, прогони реальную задачу |
| 14 | Финальный чекпоинт | Покажи полный штаб + результат |

**Тон бота:** штурман → пилот. Не обучает — ведёт.

### 2.3. Структура Спринта 3 «Оркестратор»

**Формат:** 8 шагов в Mini App. Ориентир ~7 дней, без жёсткой привязки.

| Шаг | Название в Mini App | Суть | Чекпоинт |
|-----|---------------------|------|----------|
| 1 | «Одна задача» | Выбор конкретной задачи для оркестрации | `#автоматизирую` — текст в чат |
| 2 | «Как работает оркестрация» | Обязательный ролик | — |
| 3 | «Пространство» | Создание Claude Project / ChatGPT проекта | `#моёпространство` — скриншот |
| 4 | «Штаб внутри проекта» | Загрузка тройки агентов (.md файлы) в проект или скиллы | `#штабвсборе` — скриншот |
| 5 | «Дирижёр» | Написание оркестратора (мета-инструкция цепочки) | — |
| 6 | «Проверка на прочность» | Тестовый прогон на простой задаче | — |
| 7 | «Реальная задача» | Боевой прогон с фиксацией «до/после» | — |
| 8 | «Результат» | Финальный отчёт: карточка + скриншот | `#перваяоркестрация` — карточка + скриншот |

**Бонусный материал после Шага 8:** ролик «Структура проекта» (md-файлы, категории: архитектура, дизайн, продукт, маркетинг). Не блокирует спринт.

**Два способа сборки:**
- **Способ 1 (основной):** Claude/ChatGPT Project → загрузить 3 .md файла в Project Knowledge → оркестратор в Project Instructions
- **Способ 2 (Claude Code):** Skill Creator → главный .md файл = оркестратор → папка `modules/` с 3 файлами агентов

## 3. ТРИАДА АГЕНТОВ

Базовая архитектурная единица — три роли:

| Роль | Функция | Ключевое в промпте |
|------|---------|-------------------|
| **Производитель** | Создаёт результат с нуля (текст, код, план) | Стандарты, стиль, формат |
| **Контролёр** | Получает чужой выход, проверяет по критериям, исправляет | Чеклист критериев оценки |
| **Аналитик** | Анализирует данные/паттерны, даёт рекомендации | Фокус на рычагах, не генерирует контент |

**Принцип:** сила триады — в границах ролей. Каждый агент НЕ выходит за пределы своей функции.

**Цепочка оркестрации:**

```
Аналитик → данные/контекст →
Исполнитель → результат →
Ревьюер → правки →
Исполнитель (доработка) → результат →
Ревьюер (финальное ОК или ещё один цикл)
```

**Анатомия промпта агента (4 блока):**

1. **Роль** — кто он, специализация, главная функция
2. **Контекст** — что знает о пользователе и ситуации
3. **Инструкции** — что делает и как (здесь живёт разница между ролями)
4. **Формат выхода** — в каком виде отдаёт результат

## 4. СИСТЕМА СТАТУСОВ

| Статус | Условие получения | Артефакт |
|--------|------------------|----------|
| **Сборщик** | Завершил Спринт 1 | 1 работающий агент |
| **Архитектор** 🏗️ | Завершил все 4 чекпоинта Спринта 2 | Штаб из 3 связанных агентов, карточка на Доске штабов |
| **Оператор** ⚡ | Завершил Спринт 3, показал карточку с результатом | Оркестрированный процесс, метрика «было → стало», карточка на Доске результатов |

Каждый статус — публичное признание в чате. Видимая иерархия прогресса.

**Формат карточки на Доске (Оператор):**

```
[Имя] — [профессия/ниша] — Оператор ⚡
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Задача: [что решает]
Штаб: [Роль 1] → [Роль 2] → [Роль 3]
Оркестрация: [системный промпт / скиллы]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Было: [время / качество / проблема]
Стало: [время / качество / что изменилось]
```

## 5. МЕХАНИКИ ВОВЛЕЧЕНИЯ

### 5.1. Недельный операционный цикл (~70 мин/неделю)

| День | Время | Действие | Затраты |
|------|-------|----------|---------|
| Понедельник | 10:00 | Пост Commitment | 5 мин |
| Среда | — | 10 персональных тычков | 30 мин |
| Пятница | 17:00 | Пост «Пятничная сверка» | 5 мин |
| Пятница | 18:00 | Пост «Незаменимые недели» (героизация) | 15 мин |
| Пятница | 19:00 | Обновить таблицу трекинга | 15 мин |

**Раз в 2 недели:** дайджест результатов — пост в «Объявления».

### 5.2. Героизация (каждая пятница)

- Минимум 2, максимум 4 человека за раз
- Формула: ИМЯ + ПРОФЕССИЯ + ЧТО СДЕЛАЛ + ИЗМЕРИМЫЙ РЕЗУЛЬТАТ
- Чередовать людей — не награждать одних и тех же
- Формулировка: «сделал» > «молодец». Факт > похвала

### 5.3. Понедельничный Commitment

Микро-обязательство. Порог = 1 предложение. Вопрос: «Что ты собираешься сделать с AI на этой неделе?»

### 5.4. Персональные тычки

10 штук в среду. По карте участников — адресные, с привязкой к профессии и текущему прогрессу. Если человек не отреагировал на 3 тычка подряд — прекратить.

**Эскалация:** если текстовые тычки не конвертируют (<10%) → голосовое на 30 сек.

### 5.5. Еженедельный ритуал «Пятиминутка штаба»

Постоянная механика (не привязана к спринтам). Вопрос: «Что твой штаб сделал для тебя за эту неделю? Одно предложение.»

### 5.6. Фокусы недели

Еженедельные челленджи с конкретной задачей. Фокусы с низким техническим порогом дают значительно больше участников.

### 5.7. Ремайндеры (спринты)

Если участник начал спринт, но не двигается 5+ дней — один мягкий пинг. Не преследуем.

## 7. ПРИНЦИПЫ РАБОТЫ СИСТЕМЫ

- Результат обеспечивается не контентом, а: дедлайнами, отчётами, публичностью, ограничением выбора
- Бот не обучает — ведёт (штурман → пилот)
- Один пинг при задержке, не преследуем
- Чекпоинты — в общий чат, публичность = ответственность
- Спринты без привязки к дням — каждый в своём темпе
- Сила триады — в границах ролей

### Эскалация при проблемах

| Сигнал | Диагноз | Действие |
|--------|---------|----------|
| Commitment rate < 10% | Не видят постов / формат не цепляет | Сменить время, упростить вопрос |
| Follow-through < 30% | Фокус слишком сложный | Снизить порог, формат «10 минут + скриншот» |
| Тычки < 10% конверсии | Слишком общие сообщения | Голосовые тычки |
| Активное ядро < 10% | Системная проблема продукта | Вернуться к спринт-механике, проверить первый win за 7 дней |
