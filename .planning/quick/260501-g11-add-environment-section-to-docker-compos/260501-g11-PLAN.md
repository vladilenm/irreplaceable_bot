<objective>
Add an explicit `environment:` block to `docker-compose.yml` so Timeweb App Platform's compose sanitizer injects dashboard-defined env vars into the container at runtime.

Purpose: Production currently crash-loops with `Missing required environment variable: BOT_TOKEN` (file:///app/dist/config.js:4) because Timeweb's sanitizer only forwards UI-defined env vars to the container when each var is explicitly declared via `${VAR}` interpolation inside the service's `environment:` block. Without this declaration the dashboard values are dropped before `process.env` is populated.

Output: One file modified — `docker-compose.yml` — with a new `environment:` block listing every env var consumed by `src/config.ts`, mapped through `${VAR}` (required) or `${VAR:-default}` (optional) interpolation. Local `docker compose config` resolves cleanly; production deploy on Timeweb stops crash-looping once dashboard vars are present.
</objective>

<key_facts>
- YAML-quoting rule: any value with whitespace (cron expressions, the empty default, etc.) MUST be inside a double-quoted YAML scalar so the colon/space in `${VAR:-default}` is not parsed as a YAML mapping. Use the form `KEY: "${VAR:-default}"`.
- Compose v2 reads `.env` next to `docker-compose.yml` automatically for `${VAR}` interpolation. This makes the new block work locally without modifying the override file: `.env` populates the interpolation, override file's `env_file:` still injects the same vars into the container, and `volumes:` from the override still mounts `./data`.
- NODE_ENV is also baked in via `ENV NODE_ENV=production` in Dockerfile (line 24). Declaring it in `environment:` makes it overridable from the dashboard / `.env`.
- AI_BASE_URL is the only optional var whose default is `undefined` in code. Declare as `AI_BASE_URL: "${AI_BASE_URL:-}"` so an unset var resolves to empty string (which `process.env['AI_BASE_URL']` then reads as `''`; src/config.ts assigns it directly without `??`, so empty string is acceptable — `aiBaseUrl: process.env['AI_BASE_URL']`).
- Existing comment block (lines 5-27) describing volume / env injection on Timeweb MUST be preserved verbatim — only augmented with a new paragraph about the `environment:` block.
- docker-compose.override.yml MUST NOT be modified.
</key_facts>

```yaml
      BOT_TOKEN: "${BOT_TOKEN}"
      TARGET_CHAT_ID: "${TARGET_CHAT_ID}"

      LOG_LEVEL: "${LOG_LEVEL:-info}"
      NODE_ENV: "${NODE_ENV:-production}"
      THREAD_SUMMARY_CRON: "${THREAD_SUMMARY_CRON:-30 3 * * *}"
      MESSAGE_RETENTION_DAYS: "${MESSAGE_RETENTION_DAYS:-90}"

```

- Optional defaults MUST byte-for-byte match the literals in src/config.ts: `'0 6 * * *'`, `'claude-sonnet-4-20250514'`, `'info'`, `'production'`, `'30 3 * * *'`, `90`, `'0 1 * * *'`, `'data/messages.db'`, empty string for INITIAL_TRACKED_THREAD_IDS, empty string for AI_BASE_URL (since the code reads `process.env['AI_BASE_URL']` without a fallback — empty string is the only safe representation of "absent").
