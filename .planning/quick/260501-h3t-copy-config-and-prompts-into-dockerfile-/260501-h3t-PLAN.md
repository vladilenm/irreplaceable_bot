<objective>
Fix the production Docker image so the three runtime asset reads
(`config/feeds.json`, `prompts/curator.md`, `prompts/thread-summarizer.md`)
resolve inside `/app/` instead of crashing with ENOENT at startup.

Purpose: The compiled `dist/services/*.js` resolves these paths via
`new URL('../../<path>', import.meta.url)` (compiled from `src/services/*.ts`).

so `rss.service` (line 6, module-init `readFileSync`) crashes the container
the moment `dist/index.js` imports anything that pulls in those services.

Output: A production image that contains `/app/config/feeds.json`,
`/app/prompts/curator.md`, and `/app/prompts/thread-summarizer.md`, with
the existing botuser/USER/data-ownership block untouched.

The `*.md` line is a recursive glob — it excludes EVERY `.md` file from the build
context, including `prompts/curator.md` and `prompts/thread-summarizer.md`.
A `COPY prompts ./prompts` alone would still ship an empty (or markdown-less)
`prompts/` directory. The fix requires a Docker negation rule placed AFTER
`*.md` so it overrides the broader pattern.
</dockerignore_current>

<dockerfile_current>
```dockerfile
# Build stage
FROM node:20-alpine AS builder
...
RUN npm run build

# Production stage
FROM node:20-alpine
WORKDIR /app
ENV NODE_ENV=production
COPY package.json package-lock.json ./
RUN npm ci --omit=dev
COPY --from=builder /app/dist ./dist

# v2.0 SETUP-07: pre-create /app/data with botuser ownership BEFORE USER directive.
RUN addgroup -g 1001 -S botuser && \
    adduser -S botuser -u 1001 && \
    mkdir -p /app/data && \
    chown -R botuser:botuser /app/data
USER botuser

CMD ["node", "dist/index.js"]
```

Insert the two COPY lines immediately after `COPY --from=builder /app/dist ./dist`
and BEFORE the `RUN addgroup ...` block. Do NOT add `--chown` — these are
read-only static assets, root ownership is correct (the botuser only needs
read access, which root-owned 0644 files already provide).
</dockerfile_current>
</context>
