<objective>
Заменить раздельный рендер «headline + строка → ссылка» на кликабельный заголовок: вся подстрока заголовка (вместе с emoji категории) оборачивается в <a href="url">...</a> внутри уже существующего <b>...</b>, а строка «→ url» удаляется из вывода. URL берётся из ближайшей следующей link-строки в том же блоке (до пустой строки). Если link-строки в блоке нет — заголовок остаётся `<b>headline</b>` (текущее поведение). Изменения изолированы в `digest.formatter.ts`. Промпт LLM, sender, service — НЕ трогаем (countDigestItems считает по сырому LLM-тексту до форматирования).

Purpose: Кликабельный заголовок убирает «костыль → ссылка» из визуала, делает дайджест аккуратнее и сокращает высоту сообщения, не меняя пайплайн и без риска регрессии в idempotency-метрике (countDigestItems на сыром выводе сохраняется).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output → digest.formatter | LLM может вернуть текст с literal '<', '>', '&' (например, пересказывая RSS title с HTML); это untrusted input, который превратится в `parse_mode=HTML` сообщение Telegram. |
| digest.formatter → Telegram Bot API | Сформированный HTML отправляется через sendMessageWithRetry; некорректный HTML вызовет 400 Bad Request, инъекция тегов — потенциальный XSS-эквивалент в Telegram (например, скрытые <a href="javascript:..."> — Telegram их фильтрует, но мы не должны на это полагаться). |
