<diagnostic_format>
<!-- Both error log lines must produce a msg string of this exact shape. -->
<!-- Order: action — error_code — description — chatId — threadId. Compact, dashboard-friendly. -->

First-attempt failure:
```
Telegram sendMessage failed, retrying in 3s: error_code=<code|no-code> description=<desc|err.message|String(err)> chatId=<id> threadId=<id>
```

Post-retry failure:
```
Telegram sendMessage failed after retry: error_code=<code|no-code> description=<desc|err.message|String(err)> chatId=<id> threadId=<id>
```

Extractor contract:
- Input: `unknown` err, plus chatId/threadId from params
- Output: a single string with all 4 fields
- If `err instanceof GrammyError` → use `err.error_code` and `err.description`
- Else if `err instanceof Error` → `error_code=no-code`, `description=err.message`
- Else → `error_code=no-code`, `description=String(err)` (handles `null`, `undefined`, `{}`, plain strings without crashing)
</diagnostic_format>
