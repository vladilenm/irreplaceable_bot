# Quick Task 260428-mn8: Curator-промпт arrow-формат + императив 5-7 — Summary

Восстановил рабочую связку «curator-промпт ↔ digest.formatter.ts ↔ digest.service.ts»: переписал `prompts/curator.md` под arrow-формат `→ https://...` и категорическое требование 5-7 новостей. Дайджест снова сможет публиковаться (после человеческой проверки `/dev-digest`).

## Diagnosis

**Что было сломано:** quick-fix `260428-m57` снизил порог `skipped = itemCount < 1` (с 3 до 1) и смягчил промпт до «3-5 новостей», но проблема была глубже:

- `digest.formatter.ts:33-35` (`isLinkLine`) ждёт regex `/^\s*→\s+https?:\/\//`
- `digest.service.ts:82-85` (`countDigestItems`) считает по regex `/→ https?:\/\//g`
- Текущий `prompts/curator.md:51` задавал markdown-link `[emoji] *[Заголовок](ссылка)*` — без стрелок

**Следствие:** `countDigestItems` всегда возвращал 0 → `skipped = 0 < 1 = true` → лог `No items in digest, marking as skipped` → `sendDigest` ничего не публиковал. Понижение порога с 3 до 1 не помогло, потому что итоговое число пунктов было = 0.

**Решение:** правка ТОЛЬКО `prompts/curator.md`. Код корректен, его трогать не нужно.
