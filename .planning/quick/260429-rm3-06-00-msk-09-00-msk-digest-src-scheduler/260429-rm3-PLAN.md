# Quick Task 260429-rm3 — Refresh cron MSK comments and add TZ header

## Goal
Docs-only fix. Container runs with TZ=UTC, but inline comments use MSK
(UTC+3) for human readability. The digest comment in cron.ts is off by 3h
(says 06:00 MSK, real default `0 6 * * *` renders to 09:00 MSK). `.env.example`
ships cron defaults with no TZ context, which leaves the reader to do the
UTC↔MSK math each time.

## Tasks

### Task 1 — Fix digest comment in cron.ts and add TZ note
- **files**: `src/scheduler/cron.ts`
- **action**:
  - Line 5 — change `06:00 MSK` → `09:00 MSK` for digest.
  - In the same comment block (lines 4–7), add a one-line note that
    times are MSK rendering of the default `* (UTC)` cron expressions
    (container runs TZ=UTC). Keep thread-summary `06:30 MSK` and
    retention-sweep `04:00 MSK` untouched — they already match.
- **verify**: `grep -n "09:00 MSK" src/scheduler/cron.ts` returns the
  digest line; `grep -n "06:00 MSK" src/scheduler/cron.ts` returns
  nothing.
- **done**: Comment block accurately reflects current defaults.

### Task 2 — Add TZ header to .env.example for all three cron knobs
- **files**: `.env.example`
- **action**: Add a single comment block before the cron-related
  variables explaining: container runs TZ=UTC, expressions are evaluated
  in UTC, default values render to specific MSK times. Annotate each of
  `DIGEST_CRON`, `THREAD_SUMMARY_CRON`, `RETENTION_SWEEP_CRON` with the

- **verify**: `grep -n "TZ=UTC" .env.example` returns the new header;

- **done**: Reader can see TZ context and MSK rendering without
