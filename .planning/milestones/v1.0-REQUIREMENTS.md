# Requirements: Telegram-bot "Nezamenimye"

**Defined:** 2026-04-13
**Core Value:** Participants get a quality-filtered AI digest every morning -- builds the habit and saves 30-60 minutes of daily scrolling

## v1 Requirements

### Project Setup

- [x] **SETUP-01**: Project initialized with Node.js 20+, TypeScript, Grammy, modular structure from spec
- [x] **SETUP-02**: Configuration loaded from .env (BOT_TOKEN, TARGET_CHAT_ID, AI_RADAR_THREAD_ID, DIGEST_CRON, AI_API_KEY, AI_MODEL, LOG_LEVEL)
- [x] **SETUP-03**: Logging via pino with configurable level (LOG_LEVEL)
- [x] **SETUP-04**: Dockerfile for VPS deploy (long-polling mode)

### RSS Parser

- [x] **RSS-01**: Parser fetches 9 RSS feeds (Habr, vc.ru, OpenAI, HuggingFace, LangChain, VentureBeat, Anthropic, Cursor, Tproger)
- [x] **RSS-02**: Article filtering by pubDate within last 24 hours
- [x] **RSS-03**: Fallback: if < 3 articles in 24h, expand window to 48 hours
- [x] **RSS-04**: Feed list is configurable (add/remove source without code changes)
- [x] **RSS-05**: Output as JSON array with fields: title, description, link, source, pubDate

### AI Filter

- [x] **AI-01**: Abstraction ai.service.ts supports Claude API and OpenAI API, switching via .env (AI_MODEL)
- [x] **AI-02**: System prompt for AI curator with selection criteria tuned to club context
- [x] **AI-03**: Selection of 3-5 news items from input array of 20-40 articles
- [x] **AI-04**: Quota: exactly 2 news items from vc.ru per digest (minimum 1 if insufficient worthy items)
- [x] **AI-05**: Each news item tagged with one of 6 categories (Agents, Orchestration, Models, Tools, Technologies, Business)
- [x] **AI-06**: Fallback: if < 3 significant news items, digest is not published

### Digest Delivery

- [x] **DLV-01**: Cron task launches pipeline daily at 09:00 MSK (06:00 UTC)
- [x] **DLV-02**: Published post goes to AI-radar thread via Bot API (HTML, disable_web_page_preview)
- [x] **DLV-03**: Post format: date header, 3-5 news with category emoji + title + summary + link, footer
- [x] **DLV-04**: Retry: on send error, 1 retry attempt
- [x] **DLV-05**: Idempotency: re-run on same day does not duplicate message

### Bot Commands

- [x] **CMD-01**: /start command -- welcome message with bot description and capabilities
- [x] **CMD-02**: /digest command -- manual pipeline trigger (outside schedule)
- [x] **CMD-03**: /status command -- bot status, date and result of last digest

### Reliability

- [x] **REL-01**: Graceful shutdown on SIGTERM/SIGINT (clean Grammy and cron termination)
- [x] **REL-02**: Errors are logged, do not crash bot -- bot continues after digest failure
- [x] **REL-03**: Strict TypeScript (strict: true, no any)

## v2 Requirements

### Sprint Mechanics

- **SPRINT-01**: Bot sends tasks to participants every 2-3 days per sprint timeline
- **SPRINT-02**: Pings and reminders for inactive participants (5+ days without movement)
- **SPRINT-03**: Checkpoints -- public progress reports

### Analytics

- **ANLT-01**: /analytics command -- club metrics (activity, retention, growth)
- **ANLT-02**: Digest logging to Supabase (date, article count, publication, skip reason)
- **ANLT-03**: Alert to founder if bot doesn't publish for 3 consecutive days

### Engagement

- **ENG-01**: Inline reaction buttons (Useful / Skip) on digest posts
- **ENG-02**: /summary command -- AI summarization of club chat discussions

## Out of Scope

| Feature | Reason |
|---------|--------|
| Telegram Mini App | Separate project, different stack (React + Supabase) |
| Webhook integrations | Not needed for MVP, VPS + long-polling sufficient |
| Product dashboard | Separate service, not part of bot |
| OAuth / user auth | Bot works in group, no separate auth needed |
| Extended RSS sources (Google AI, The Verge, smol.ai) | Add after MVP validation |
| Supabase for logging | MVP -- pino logs only, DB later |
