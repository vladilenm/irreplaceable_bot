**1. Create `src/modules/digest/digest.formatter.ts`** (per D-09, D-10)

Export function `formatDigestHtml(plainText: string): string` that converts LLM plain text output to Telegram HTML.

The LLM output structure (from curator prompt in docs/rss.md):
```
📡 AI-радар | 5 апреля

🧠 Claude Opus 4.6 — Anthropic обновила флагман
Новая модель показывает прирост...
→ https://anthropic.com/news/...

💰 Яндекс запустил маркетплейс...
Компании могут подключать...
→ https://vc.ru/...

———
Дайджест Клуба Незаменимых
Система > Навык
```

Transformation rules:
- First escape all HTML entities in the entire text (`&` -> `&amp;`, `<` -> `&lt;`, `>` -> `&gt;`)
- Wrap the header line (starts with `📡`) in `<b>...</b>` tags
- For each news item: wrap the headline line (starts with category emoji like 🧠, 🔗, 🤖, 🛠, ⚡, 💰) in `<b>...</b>` tags
- Convert `→ URL` lines: replace `→ https://...` with `→ <a href="https://...">ссылка</a>` (the URL was already escaped, so un-escape the `&amp;` back to `&` inside href attribute values only)
- Keep the footer separator `———` and footer text as-is
- Return the resulting HTML string

Use regex-based approach. No external libraries needed.

**Replace the stub `src/scheduler/cron.ts`** with a working node-cron scheduler (per D-03, D-04, D-05).

Implementation:

```typescript
import cron from 'node-cron';
import type { ScheduledTask } from 'node-cron';
import { config } from '../config.js';
import { logger } from '../utils/logger.js';
import { runDigestPipeline } from '../modules/digest/digest.service.js';
import { sendDigest } from '../modules/digest/digest.sender.js';

let task: ScheduledTask | null = null;

export function startScheduler(): void {
  const cronExpression = config.digestCron; // Default: "0 6 * * *" (06:00 UTC = 09:00 MSK)

  if (!cron.validate(cronExpression)) {
    logger.error({ cronExpression }, 'Invalid cron expression, scheduler not started');
    return;
  }

  task = cron.schedule(cronExpression, async () => {
    logger.info('Cron triggered: starting digest pipeline');
    try {
      const result = await runDigestPipeline();
      if (result.alreadyPublished) {
        logger.warn('Cron: digest already published today, skipping send');
        return;
      }
      await sendDigest(result);
      logger.info({ itemCount: result.itemCount, skipped: result.skipped }, 'Cron: digest cycle complete');
    } catch (err: unknown) {
      logger.error({ err }, 'Cron: digest cycle failed');
    }
  });

  logger.info({ cronExpression }, 'Scheduler started');
}

export function stopScheduler(): void {
  if (task) {
    task.stop();
    task = null;
    logger.info('Scheduler stopped');
  } else {
    logger.debug('Scheduler: no active task to stop');
  }
}
```

Key points:
- Uses `node-cron` package (already in package.json dependencies)
- Validates cron expression before scheduling
- Cron handler is async: runs pipeline, checks alreadyPublished, sends digest
- Errors in cron handler are caught and logged (don't crash the bot, per REL-02)
- `stopScheduler()` calls `task.stop()` for clean shutdown (per REL-01)
- Logs cron start, each trigger, and stop events (per D-05)
- Import type `ScheduledTask` from `node-cron` for the task variable typing

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> HTML formatter | Untrusted text from LLM crosses into Telegram HTML message |
| Bot -> Telegram API | Outbound API call with credentials |
| Cron -> Pipeline | Scheduled trigger initiates network calls (RSS + LLM + Telegram) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-01 | Spoofing | digest.formatter.ts | mitigate | HTML-escape all LLM output before wrapping in tags -- prevents injecting arbitrary HTML/script via crafted RSS titles |
| T-03-02 | Tampering | state.json | accept | state.json is local file on server, no external write access; corrupted state handled by readState() defaults (from Phase 2) |
