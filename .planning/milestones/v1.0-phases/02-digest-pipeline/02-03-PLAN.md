State file path: `new URL('../../data/state.json', import.meta.url)` — but since this file is relative to project root, use a more robust approach:

```typescript
const STATE_PATH = new URL('../../../data/state.json', import.meta.url);
```
(From src/modules/digest/ we need to go up 3 levels to project root, then into data/)

Helper `readState(): PipelineState`:
- Try readFileSync(STATE_PATH, 'utf-8'), parse JSON
- On error (file missing), return default `{ lastDigestDate: null, lastSkipped: false }`

Helper `writeState(state: PipelineState): void`:
- Ensure data/ directory exists with mkdirSync (recursive: true)
- writeFileSync(STATE_PATH, JSON.stringify(state, null, 2))

Helper `countDigestItems(text: string): number`:
- Count occurrences of the link arrow pattern "→ http" in the LLM output text
- This is how we determine itemCount from the ready-made text (per D-07/D-10 the LLM returns formatted text, not JSON)
- Return the count

Export async function `runDigestPipeline(): Promise<DigestResult>`:

```
1. Read state
2. Determine hoursBack:
   - Default: 24
   - If state.lastSkipped is true, use 48 (per D-13 fallback logic)
   - Log the window being used
3. Call fetchFeeds(hoursBack)
4. If articles.length === 0:
   - Log warning "No articles found in {hoursBack}h window"
   - Write state: { lastDigestDate: new Date().toISOString(), lastSkipped: true }
   - Return { text: '', itemCount: 0, skipped: true, date: new Date() }
5. Log: "Fetched {n} articles, sending to AI filter"
6. Call filterArticles(articles)
7. Count items in response: countDigestItems(text)
8. Determine skip (per D-14, AI-06):
   - If itemCount < 3, set skipped = true
   - Log accordingly
9. Write state:
   - lastDigestDate: new Date().toISOString()
   - lastSkipped: skipped
10. Return DigestResult { text, itemCount, skipped, date: new Date() }
```

Log at each step with structured data. Use ESM import paths with `.js` extensions. No `any` types.
